# TinyBrain v0.1.0 Release Notes

**Release Date:** October 25, 2025  
**Codename:** "Foundation"  
**Status:** MVP Complete (TB-001 through TB-007)

---

## 🎉 What's New

TinyBrain v0.1.0 is the first public release of a **Swift-native, on-device LLM runtime** for iOS and macOS. This release delivers the core infrastructure for running large language models entirely on-device with a focus on educational transparency and practical performance.

---

## ✨ Features

### Core Runtime (TB-001, TB-002)

- **Swift-Native Tensor Engine**
  - Generic `Tensor<Element>` supporting Float32, Float16, Int8
  - Copy-on-write optimization for memory efficiency
  - Accelerate framework integration (BLAS, vDSP)
  - Operations: MatMul, Softmax, LayerNorm, GELU, element-wise ops
  - **179 passing tests** (TDD methodology)

### Metal GPU Acceleration (TB-003, TB-004)

- **Tiled MatMul Kernel**
  - 16×16 threadgroups with shared memory
  - Automatic CPU/GPU backend selection
  - Buffer pooling (450× faster allocation)
  - GPU-resident tensors (eliminate transfer overhead)
  
- **Performance Reality Check**
  - M4 Max has AMX (Apple Matrix Extension) matrix coprocessor
  - CPU (AMX) competitive with or beats GPU for single matmul (0.7-1.3×)
  - GPU wins for batched workflows (2-4× for chained operations)
  - See `docs/TB-004-M4-FINDINGS.md` for honest assessment

### INT8 Quantization (TB-004)

- **Per-Channel Symmetric Quantization**
  - 75% memory reduction (4.4GB → 1.1GB for TinyLlama)
  - <1% perplexity delta vs Float32
  - BLEU score: 0.92 (92% similarity)
  - Quality regression tests validate accuracy

### Paged KV-Cache (TB-004)

- **Efficient Attention State Management**
  - 2048-token context window
  - 16-token pages with pre-allocation
  - Zero-copy page reuse
  - 10,000 cycle stress test (no memory leaks)

### Tokenization & Sampling (TB-005)

- **BPE Tokenizer**
  - Full Byte Pair Encoding implementation
  - Unicode NFC normalization
  - Special tokens (BOS, EOS, UNK, PAD)
  - Multilingual support (accents, emoji)
  - 17 passing tests

- **Advanced Sampling Strategies**
  - Greedy (deterministic)
  - Temperature scaling (0.1-2.0)
  - Top-K filtering
  - Top-P (Nucleus) sampling
  - Repetition penalty
  - 19 passing tests

### Streaming Runtime (TB-004, TB-005)

- **AsyncSequence-Based Generation**
  - Real-time token streaming
  - SwiftUI integration
  - <150ms latency per token
  - Configurable generation parameters
  - Stop token support

### SwiftUI Demo App (TB-006)

- **TinyBrain Chat**
  - Clean MVVM architecture
  - Real-time token streaming with typewriter animation
  - Telemetry display (tokens/sec, memory, probability)
  - Message history
  - Copy functionality
  - Accessibility labels
  - **53 passing tests**

### Benchmark Harness (TB-007)

- **Production-Grade Benchmarking**
  - YAML scenario loading
  - JSON output for CI/automation
  - Markdown table generation
  - Memory tracking
  - Device info reporting
  - Warmup iterations
  - **9 passing tests**

### Model Conversion (TB-007)

- **PyTorch → TBF Converter**
  - Load PyTorch checkpoints (`.pt`, `.safetensors`)
  - Auto-detect model architecture
  - INT8 quantization pipeline
  - TBF binary format writer
  - Progress bars and error handling
  - **7 passing Python tests**

## 🚀 Installation

### Prerequisites

- **macOS:** 14 Sonoma or later
- **iOS:** 17 or later
- **Xcode:** 16+
- **Swift:** 5.10+
- **Hardware:** Apple Silicon recommended (M1/M2/M3/M4)

### Quick Start

```bash
# Clone repository
git clone https://github.com/vivekptnk/tinybrain.git
cd tinybrain

# Setup (install dependencies, build, test)
make setup

# Run demo
swift run tinybrain-chat
```

### Xcode Workflow (Recommended for ChatDemo)

```bash
# Open in Xcode
open Package.swift

# Select ChatDemo scheme
# Edit Scheme → Run → Options → Uncheck "Use the sandbox"
# (Fixes TextField input on macOS 15.x)

# Run
Cmd+R
```

### Python Environment (for model conversion)

```bash
# Create virtual environment
python3 -m venv .venv
source .venv/bin/activate

# Install dependencies
pip install -r requirements.txt

# Convert model
python Scripts/convert_model.py \
  --input model.pt \
  --output Models/model.tbf \
  --quantize int8
```

## ⚠️ Known Issues

### macOS TextField Input (Workaround Available)

**Problem:** SwiftUI TextField doesn't accept keyboard input in SPM executables on macOS 15.x Tahoe

**Workaround:**
1. Open `Package.swift` in Xcode
2. Edit Scheme → Run → Options
3. Uncheck "Use the sandbox"
4. Run the app

**Impact:** macOS ChatDemo only  
**iOS:** Not affected  
**Status:** Documented workaround, permanent fix deferred to future Xcode project conversion

### AMX vs GPU Performance

**Finding:** M4 Max CPU (AMX) competitive with or beats GPU for single matmul operations

**Why:** Apple Matrix Extension (AMX) is a dedicated matrix coprocessor that Accelerate uses automatically

**Impact:** Original 3-8× GPU speedup target adjusted to reality  
**Current:** 0.7-1.3× competitive, GPU wins for batched workloads

**Status:** Expected behavior, documented in `docs/TB-004-M4-FINDINGS.md`

### TBF Model Loading

**Status:** Converter complete, ModelRunner integration pending

**Current:** Can convert PyTorch → TBF, but ModelRunner uses toy models

**Next:** TB-008 will implement TBF loader in ModelRunner

**Workaround:** Use toy model generator for testing infrastructure
