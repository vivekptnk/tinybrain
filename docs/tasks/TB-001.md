# TB-001 – Scaffold TinyBrain Workspace

## Objective
Lay down the foundational project structure, tooling, and documentation so contributors can build, run, and test TinyBrain consistently on macOS/iOS targets.

## Background
The PRD specifies a Swift-native runtime with multiple modules (runtime, Metal, demos, docs). Before feature work, the repo must expose predictable entry points (`Package.swift`, `Sources/`, `Examples/`, `Tests/`), automated formatting/linting, and initial docs. This task unblocks every downstream feature team and ensures CI parity with Apple Silicon targets.

## Work Items
1. Create Swift Package Manager workspace with targets: `TinyBrainRuntime`, `TinyBrainMetal`, `TinyBrainTokenizer`, `TinyBrainDemo`, and matching test targets.
2. Scaffold `/Sources`, `/Examples/ChatDemo`, `/Tests`, `/Scripts`, and `/docs/overview.md` as referenced in the PRD.
3. Author `Package.swift` with platform constraints (iOS 17, macOS 14) and dependencies (Swift ArgumentParser placeholder).
4. Configure SwiftFormat/SwiftLint (or Swift compiler plugins) plus a Makefile or Swift Package command alias for `lint`, `test`, `generate-docs`.
5. Create `README.md` describing goals, local dev setup, and quick start instructions from the PRD.
6. Add a GitHub Actions workflow (or equivalent CI) that runs lint + test on macOS 14 with Xcode 16. Use matrix jobs for macOS/iOS destinations if feasible.
7. Seed DocC documentation catalog (`Docs/TinyBrain.docc`) and ensure `swift package generate-documentation` succeeds locally.

## Acceptance Criteria
- Running `swift build` and `swift test` from repo root succeeds on Apple Silicon without manual tweaks.
- CI workflow executes lint + test on macOS 14/Xcode 16 and displays status badge placeholder in `README.md`.
- Repository contains structured directories (`Sources/`, `Examples/ChatDemo`, `Tests/`, `docs/overview.md`) and DocC catalog checked in.
- `README.md` covers project overview, prerequisites, quick start, and contribution guide sections consistent with PRD terminology.
- Tooling commands (`make lint`, `make test`, or documented equivalents) run end-to-end locally.

## Dependencies
None. This is the first task and unlocks all others.
