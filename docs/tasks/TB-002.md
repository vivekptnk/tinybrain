# TB-002 – Build Swift Tensor Engine MVP

## Objective
Implement the core Swift tensor runtime that handles memory, math primitives, and CPU fallbacks for transformer inference. This layer should be pedagogical yet performant enough to validate kernels before Metal acceleration.

## Background
Section 4 of the PRD lists the “Swift Tensor Engine” as P0. It must expose tensor shapes, dtypes, broadcasting, and differentiable building blocks used by attention/MLP layers. Having a solid CPU-backed engine simplifies debugging and unit testing before GPU routing (Task TB-003).

## Work Items

### Core Implementation (TB-002 Scope)
1. Extend `Tensor` struct to support Float32 operations with basic contiguous storage (stride-aware storage **deferred to TB-003**).
2. Implement core ops via Accelerate framework: matmul (cblas_sgemm), elementwise add/mul (vDSP_vadd/vmul), GELU, LayerNorm, Softmax with CPU fallback paths.
3. Provide tensor initialization utilities (zeros, filled, random normal, identity/eye) using vDSP for vectorized initialization where possible.
4. Add subscript access (`tensor[i, j]`) for reading/writing individual elements with bounds checking.
5. Implement comprehensive **TDD tests** that validate numerical accuracy (relative error < 1e-5 for Float32) and serve as educational examples.
6. Document operations in DocC articles explaining each operation's purpose in LLM inference (e.g., "Why MatMul?", "What is GELU?").
7. Create simple benchmark script (`Scripts/benchmark-ops.swift`) measuring CPU op throughput and comparing to manual implementations.

### Deferred to Later Tasks
- **Generic types** (Float16, Int8) → **TB-004** (quantization task naturally requires this)
- **Copy-on-write optimization** → **TB-004** (premature optimization; get correctness first)
- **Stride-aware storage** (transpose without copy) → **TB-003** (Metal needs this for efficient kernels)
- **mmap-backed loading** → **TB-004** (quantization + model loading task)
- **Op registry abstraction** → **TB-003** (CPU/GPU selection when Metal is ready)

## Acceptance Criteria
- `TinyBrainRuntime` target exposes public APIs for tensor creation, inspection, and the specified ops with doc comments.
- Unit tests in `Tests/TinyBrainRuntimeTests` validate numeric accuracy (e.g., matmul relative error < 1e-4 for FP16/FP32 comparisons).
- Benchmark script (simple) reports baseline tokens/sec for CPU-only inference on a toy model and is checked into `/Scripts`.
- Documentation updates describe how tensors flow through the runtime and link to DocC articles.
- No Metal-specific dependencies are introduced yet; code compiles on simulator targets.
