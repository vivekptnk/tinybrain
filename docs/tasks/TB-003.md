# TB-003 – Implement Metal Kernel Suite

## Objective
Deliver GPU-accelerated kernels for the highest-impact transformer ops (MatMul, Softmax, LayerNorm, Quant/Dequant) and integrate them into the runtime’s backend abstraction so workloads automatically route to Metal on supported devices.

## Background
TinyBrain’s differentiation is a Swift-native stack with performant Metal kernels (PRD Sections 4 & 7). This task operationalizes that claim by pairing Metal shader code with Swift bindings, ensuring kernels interoperate with the tensor engine built in TB-002.

## Work Items
1. Create a `TinyBrainMetal` target containing `.metal` shader files for FP16 MatMul, fused bias+activation, INT8 dequantization, and paged KV-cache updates.
2. Build a command encoder pipeline that reuses shared buffers and minimizes data copies; expose a Swift API like `MetalBackend.execute(_ op: BackendOp)`.
3. Implement kernel autotuning (tile sizes, threadgroups) for iPhone 15 Pro, iPad M2, and Mac M3 profiles; persist defaults in a JSON config.
4. Add fallback logic: if a kernel/device combo is unsupported, route back to CPU ops transparently with logging hooks.
5. **[Deferred from TB-002]** Implement stride-aware tensor operations to support transpose/reshape without data copies; update Metal kernels to handle non-contiguous tensors.
6. Write benchmarking utilities comparing CPU vs Metal throughput for representative tensor sizes; surface metrics (ms/token, tokens/sec, energy) in console output.
7. Include validation tests that run lightweight kernels on CI (e.g., macOS destination) to confirm determinism and numeric tolerances.
8. Document kernel architecture, buffer layout, tuning strategy, and stride handling in `docs/overview.md` + DocC articles.

## Acceptance Criteria
- MatMul + Softmax + LayerNorm + Quant/Dequant Metal kernels compile and run on macOS 14 and iOS 17 simulators/devices.
- Runtime automatically selects Metal backend when available; API consumers do not reference Metal types directly.
- Benchmarks show ≥3× speedup over CPU baseline for 2k×2k MatMul on Mac M3 (report stored under `/benchmark-results/metal-vs-cpu.md`).
- Unit/integration tests validate parity between Metal and CPU outputs within tolerance (<1e-3 for FP16 operations).
- Documentation describes how to enable/disable Metal backend, lists supported devices, and cites tuning data.

## Dependencies
- TB-001 (scaffold) and TB-002 (tensor engine) completed.
