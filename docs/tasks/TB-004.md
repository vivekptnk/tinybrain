# TB-004 – INT8 Quantization & Paged KV Cache

## Objective
Implement per-channel INT8 quantization (v1 target) and the paged KV-cache memory manager so TinyBrain can stream tokens efficiently on-device with constrained RAM.

## Background
The PRD’s P0/P1 goals call for INT8 quantization, paged KV-cache, and streaming token generation. With Metal kernels in place, the runtime now needs compact weight formats and reusable attention state buffers to hit latency/energy targets.

## Work Items
1. **[Deferred from TB-002]** Refactor `Tensor` to use generic types (`Tensor<Element: TensorElement>`) supporting Float32, Float16, and Int8; implement copy-on-write optimization with `isKnownUniquelyReferenced` for efficient memory management.
2. Design quantized weight container formats (metadata + INT8 blobs) compatible with mmap loading; document schema for future converter tooling.
3. Implement calibration utilities that compute per-channel scales/zero-points from FP16 checkpoints, supporting symmetric/asymmetric modes.
4. Extend tensor ops to handle quantized inputs, including efficient dequantization paths interfacing with Metal kernels from TB-003.
5. Build a paged KV-cache allocator that pre-reserves GPU/CPU memory, supports context window 2048, and evicts/compacts pages as sequences grow.
6. Add streaming-friendly APIs (`ModelRunner.step(token:)`) that reuse KV pages instead of recomputing history.
7. Write regression tests verifying numerical fidelity vs FP16 reference (bleu/perplexity on sample prompts) and memory leak tests.
8. Update docs with diagrams showing KV page flow, quantization pipeline, CoW optimization strategy, and configuration flags.

## Acceptance Criteria
- INT8 weight loader ingests sample TinyLlama checkpoint and produces tensors consumable by runtime without conversions at call sites.
- End-to-end inference on a 1.1B TinyLlama INT8 model fits within target memory budget for iPhone 15 Pro (≤6 GB peak) while streaming 2048 tokens.
- KV-cache API exposes page stats (hits, evictions) and passes stress tests for 4 simultaneous streams.
- Documentation includes a troubleshooting section covering quant artifacts and KV-cache tuning knobs.
- Benchmarks show ≥35% memory savings vs FP16 baseline with no more than 1% drop in benchmark accuracy metrics.
