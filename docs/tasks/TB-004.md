# TB-004 – INT8 Quantization & Paged KV Cache

## Objective
Implement per-channel INT8 quantization (v1 target) and the paged KV-cache memory manager so TinyBrain can stream tokens efficiently on-device with constrained RAM.

## Background
The PRD’s P0/P1 goals call for INT8 quantization, paged KV-cache, and streaming token generation. With Metal kernels in place, the runtime now needs compact weight formats and reusable attention state buffers to hit latency/energy targets.

## Work Items

### High Priority: Fix Metal Performance (From TB-003)
1. **[Critical from TB-003]** Implement persistent GPU buffers to eliminate per-operation transfer overhead:
   - GPU-resident tensor type that keeps data on Metal buffers
   - Lazy CPU↔GPU synchronization (only when needed)
   - Batch multiple operations before transferring results back
   - Target: Achieve originally promised 3-8× Metal speedup

### Core Quantization & Memory Work
2. **[Deferred from TB-002]** Refactor `Tensor` to use generic types (`Tensor<Element: TensorElement>`) supporting Float32, Float16, and Int8; implement copy-on-write optimization with `isKnownUniquelyReferenced` for efficient memory management.
3. Design quantized weight container formats (metadata + INT8 blobs) compatible with mmap loading; document schema for future converter tooling.
4. Implement calibration utilities that compute per-channel scales/zero-points from FP16 checkpoints, supporting symmetric/asymmetric modes.
5. Extend tensor ops to handle quantized inputs, including efficient dequantization paths interfacing with Metal kernels from TB-003.
6. Build a paged KV-cache allocator that pre-reserves GPU/CPU memory, supports context window 2048, and evicts/compacts pages as sequences grow.
7. Add streaming-friendly APIs (`ModelRunner.step(token:)`) that reuse KV pages instead of recomputing history.
8. Write regression tests verifying numerical fidelity vs FP16 reference (bleu/perplexity on sample prompts) and memory leak tests.
9. Update docs with diagrams showing KV page flow, quantization pipeline, CoW optimization strategy, GPU-resident tensor architecture, and configuration flags.
10. **[Deferred from TB-003]** Optionally add GPU kernels for Softmax/LayerNorm/GELU if profiling shows they are performance bottlenecks (current CPU implementations are sufficient).
11. **Re-benchmark Metal** after persistent buffers are implemented to validate 3-8× speedup is achieved.

### Deferred to Even Later Tasks
- **Device-specific kernel autotuning** → **TB-007** (benchmarking/optimization phase)
- **Fused kernel operations** (bias+activation, attention+softmax) → **TB-005+** (advanced optimization)

## Acceptance Criteria
- INT8 weight loader ingests sample TinyLlama checkpoint and produces tensors consumable by runtime without conversions at call sites.
- End-to-end inference on a 1.1B TinyLlama INT8 model fits within target memory budget for iPhone 15 Pro (≤6 GB peak) while streaming 2048 tokens.
- KV-cache API exposes page stats (hits, evictions) and passes stress tests for 4 simultaneous streams.
- Documentation includes a troubleshooting section covering quant artifacts and KV-cache tuning knobs.
- Benchmarks show ≥35% memory savings vs FP16 baseline with no more than 1% drop in benchmark accuracy metrics.
