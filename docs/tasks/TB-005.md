# TB-005 – Tokenizer, Sampler, and Streaming Runtime API

## Objective
Deliver the language-facing components—SentencePiece/BPE tokenizer, configurable sampler (top-k, top-p, temperature), and streaming output interfaces (AsyncSequence/Combine)—so TinyBrain can power interactive chat experiences.

## Background
PRD Sections 4, 7, and 14 emphasize educational clarity plus a SwiftUI chat app with streamed tokens. With core inference stack ready, we now need ergonomic APIs that convert text to tokens, drive decoding loops, and emit incremental responses.

## Work Items
1. Implement a tokenizer module that can ingest SentencePiece model files, support special tokens, and run entirely in Swift without Python bindings.
2. Build detokenization utilities with Unicode normalization safeguards and unit tests covering multilingual prompts.
3. Create a sampler service exposing strategies (greedy, temperature, top-k, top-p, repetition penalty) configurable per request.
4. Design a streaming interface leveraging Swift Concurrency (`AsyncStream`) and Combine publishers to push tokens/logits to UI/CLI clients.
5. Add logging hooks for token probabilities and energy estimates (placeholders) as described in the PRD overlay specs.
6. Provide examples under `Examples/ChatDemo` that show how to wire tokenizer + sampler + runtime for a TinyLlama checkpoint.
7. Document API usage with quick-start snippets and highlight extensibility points for custom token filters.

## Acceptance Criteria
- Tokenizer achieves parity with reference SentencePiece outputs on provided vocab fixtures (exact token IDs) and passes regression tests.
- Streaming API emits tokens with sub-150 ms latency between backend callback and consumer on Mac M2 (simulated load test provided).
- Sampler supports runtime configuration overrides and exposes telemetry (selected strategy, entropy) for debugging.
- Example app compiles and demonstrates streaming text in the console or SwiftUI preview.
- Docs include end-to-end “Prompt → Tokens → Stream” tutorial.

## Dependencies
- TB-001 through TB-004 completed (runtime, Metal, quantization, KV cache).

## Notes & References
- Follow Combine best practices to avoid main-thread blocking in SwiftUI.
- Consider protocol-based tokenizer abstraction to accommodate future SentencePiece vs BPE models.
