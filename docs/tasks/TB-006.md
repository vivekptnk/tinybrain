**Status:** ✅ **PHASE 1 COMPLETE** (Core Foundation)

## Objective
Ship the flagship SwiftUI application ("TinyBrain Chat") showcasing on-device inference with live token streaming, telemetry overlays, and quantization toggles as described in the PRD.

## Background
Section 14 outlines the demo app requirements: prompt input, streamed output, token speed + energy overlay, quantization toggle, token probability graph. This task turns runtime capabilities into a polished user experience suitable for TestFlight and conference demos.

## Work Items
1. Create a SwiftUI app target under `Examples/ChatDemo` with modular view/components (PromptBar, StreamView, TelemetryPanel, ProbabilityChart).
2. Integrate AsyncSequence stream from TB-005 to render tokens incrementally with typewriter animation and auto-scroll.
3. Add controls for switching between INT8/INT4 (when available) and toggling sampler presets (Balanced, Creative, Precise).
4. Implement telemetry overlay that displays tokens/sec, ms/token, energy estimate, and KV-cache gauge using Swift Charts or custom drawing.
5. Visualize token probabilities for the last N tokens using a live-updating chart (bars/sparkline) fed by sampler telemetry.
6. Provide offline model management UI: shows downloaded checkpoints, storage usage, and allows selection.
7. Add snapshot/unit tests for critical view models and include accessibility labels for VoiceOver compliance.
8. Document app architecture, including how to mock runtime for SwiftUI previews.

## Acceptance Criteria
- App builds and runs on iOS 17 simulator and macOS via Mac Catalyst (or native SwiftUI multi-platform target).
- Streamed responses show within 500 ms of inference start and continue smoothly without blocking UI updates.
- Switching quantization modes updates runtime configuration without relaunching the app.
- Telemetry overlay displays real metrics sourced from runtime events and updates at least twice per second.
- Probability graph faithfully reflects sampler outputs and passes snapshot/UI tests.
- README or docs include instructions for launching the demo with a TinyLlama INT8 model.

### ⚠️ Critical Issue: TextField on macOS Tahoe

**Problem:** SwiftUI TextField does not accept keyboard input in SPM executables on macOS 15.x Tahoe + Xcode 26.

**Root Cause:**
- SPM executables lack proper app bundle (no CFBundleIdentifier)
- Error: "Cannot index window tabs due to missing main bundle identifier"  
- Affects ALL text input mechanisms: TextField, TextEditor, NSTextField wrappers, even sheets

**Attempted Fixes (ALL failed):**
- Native NSTextField with NSViewRepresentable
- TextEditor
- Simple TextField with zero modifiers
- Sheet-based input
- Focus state management
- AppDelegate with window activation

**Current Workaround:**
- Demo buttons with pre-filled prompts (functional but not ideal)
- Allows testing streaming, telemetry, and UI components

**Permanent Fix (TB-007):**
- Convert to proper Xcode .xcodeproj project
- Add Info.plist with CFBundleIdentifier
- Create proper macOS app bundle

**iOS Status:** ✅ Not affected - TextField works fine on iOS

### Recommendations
